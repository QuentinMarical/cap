# Code of Conduct

## ü§ù Our Pledge

We are committed to providing a welcoming, friendly, and harassment-free environment for everyone contributing to or using the Carter Addon Pack, regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

## üìú Our Standards

### ‚úÖ Examples of Positive Behavior

- **Be Respectful**: Treat everyone with respect and consideration
- **Be Constructive**: Provide helpful feedback and suggestions
- **Be Collaborative**: Work together toward common goals
- **Be Welcoming**: Help newcomers feel included
- **Be Patient**: Remember everyone was a beginner once
- **Be Professional**: Keep discussions focused and on-topic
- **Give Credit**: Acknowledge others' contributions and ideas

### ‚ùå Unacceptable Behavior

- Harassment, discrimination, or offensive comments
- Personal attacks, insults, or trolling
- Publishing others' private information without permission
- Sexual or violent content, imagery, or language
- Spam, advertising, or self-promotion
- Deliberate intimidation or stalking
- Disrupting discussions or derailing conversations
- Any conduct that could reasonably be considered inappropriate

## üõ°Ô∏è Scope

This Code of Conduct applies to:
- All project spaces (GitHub, Discord, etc.)
- Issue discussions and pull requests
- Code reviews and comments
- Community forums and chat channels
- Official project events
- Representing the project publicly

## üì¢ Reporting

If you experience or witness unacceptable behavior:

1. **Document it**: Note what happened, when, and who was involved
2. **Report it**: Contact the project maintainers privately
3. **Be specific**: Provide context and evidence if possible

**How to report:**
- GitHub: Private message to repository maintainers
- Email: Use contact information from maintainer's GitHub profile

**What happens next:**
- Reports will be reviewed promptly and confidentially
- Appropriate action will be taken based on the situation
- Reporter's privacy will be protected
- Both parties may be contacted for additional information

## üî® Enforcement

Project maintainers have the right and responsibility to:

- Remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct
- Ban temporarily or permanently any contributor for behaviors they deem inappropriate, threatening, offensive, or harmful
- Take appropriate action for any behavior that violates this Code of Conduct

### Consequences

Violations may result in:

1. **Warning**: First offense, minor issue
2. **Temporary Ban**: Repeated or more serious violations
3. **Permanent Ban**: Severe or persistent violations

## üéØ Guidelines for Healthy Discussions

### Reporting Bugs
- Be clear and specific about the problem
- Include reproduction steps and system information
- Don't blame or criticize the developers
- Check if the issue was already reported

### Suggesting Features
- Explain the use case and benefits
- Be open to feedback and alternatives
- Understand not all suggestions can be implemented
- Respect maintainers' decisions on project direction

### Contributing Code
- Follow the project's coding standards
- Write clear commit messages
- Test your changes thoroughly
- Be receptive to code review feedback
- Don't take criticism personally

### Helping Others
- Be patient with beginners
- Provide helpful, constructive feedback
- Link to relevant documentation
- Encourage learning and experimentation

## üí¨ Communication Guidelines

- **Use clear language**: Write in a way everyone can understand
- **Stay on topic**: Keep discussions relevant to CAP
- **Be concise**: Respect others' time
- **Use appropriate channels**: Bugs in issues, questions in discussions
- **Search first**: Check if your question was already answered
- **Provide context**: Include relevant details when asking for help

## üåç Diversity and Inclusion

We value diversity and strive to create an inclusive environment where:

- All skill levels are welcomed (from beginners to experts)
- People from all backgrounds feel comfortable participating
- Different perspectives and ideas are valued
- Language barriers are acknowledged and accommodated when possible
- Mistakes are seen as learning opportunities
